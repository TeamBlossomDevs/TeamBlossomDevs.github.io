# The GitHub Label Cleanup Proposal
Labels are important. Our repo uses them to categorize issues and pull requests (PRs, for short) by their status, type, topic, and other information. <br> For example, an issue/PR labeled `status: pending triage` hasn‚Äôt been reviewed, while an issue/PR labeled `status: rejected` won‚Äôt be added to the game. 

Over the course of 2024, there was a growing labeling problem. Many issues had incorrect labels or no labels at all, the labels themselves were cluttered and inconsistent, and we had a bot running around marking things as ‚Äú[potential duplicate](https://github.com/FunkinCrew/Funkin/issues/3942#issuecomment-2549923510)‚Äù (it was wrong every time).

That‚Äôs why I opened [The GitHub Label Cleanup Proposal](https://github.com/FunkinCrew/Funkin/issues/3920), which detailed exactly how to clean up and organize the labels. Delete some here, create some there, group these together, change those colors, etc. etc. 

It was a lot!

After some discussion and revision, Eric implemented all of the changes from the proposal.

The difference was night and day: the new labels were neatly categorized, colored consistently, and clearly defined. My favorite feature is the use of green and red to color labels indicating completion. You can now easily tell apart which issues are `status: resolved` (green) and which ones are `status: rejected` (red). How convenient!

With these huge changes, AbnormalPoof and I have made sure to label every issue and PR created since Weekend 1 released (almost 2000 in total). No more mystery issues :)
